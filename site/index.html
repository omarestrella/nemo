Keep production close without keeping a terminal open.

Install

A Mac app, a host agent, and a pairing flow.

Install Nemo on your Mac, install the agent on the server, then pair them once from the menu bar. The app stores its credential in Keychain and talks to the agent over the endpoint you choose.

macOS app

Install the signed release build with the Homebrew cask from the Nemo repo.

brew tap omarestrella/nemo https://github.com/omarestrella/nemo
brew install --cask omarestrella/nemo/nemo

Linux host agent

Run the installer on the server to install the agent, service account, and systemd unit.

curl -fsSL https://omarestrella.github.io/nemo/install.sh | sh

Workflow

Built for quick checks, not a new dashboard habit.

Scan the current state.

See which apps are running, which URLs are attached, and whether HTTPS is configured before you reach for a shell.

Open the context you need.

Jump from the menu bar into app URLs, logs, recent platform events, or the host connection settings without digging through command history.

Pair from the browser.

Start from Nemo, approve the request on the host, and let the Mac app finish the exchange. No long-lived setup token needs to be copied into notes.

Security model

Read-only by default.

The long-running service runs as a dedicated nemo-agent user. Root is only needed for install and repair, and server commands go through a constrained wrapper. Paired clients use bearer credentials, and untrusted remote access should go through HTTPS or a private network transport.
